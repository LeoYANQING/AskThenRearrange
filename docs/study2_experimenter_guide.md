# PrefQuest Study 2 实验员操作指南

**版本**：2.0（2026-04-20）  
**适用对象**：主实验员（PI）及助手实验员  
**实验设计**：被试内设计，3 种策略条件（DQ / UPF / PAR），每人 3 个 trial  
**预计时长**：每次 session 约 65–75 分钟

**角色分工**

| 角色 | 职责 |
|---|---|
| **主实验员（PI）** | 接待参与者、主导口头介绍、主导退出访谈、处理突发情况、负责数据导出 |
| **助手实验员** | 操作实验员控制面板、控制 STT 录音、推进 Phase、做访谈笔记 |

> Phase 2 对话期间：助手专注操作屏幕，主实验员坐在参与者旁边观察，**不干预对话内容**。

---

## 一、实验前

### 1.1 设备清单

- [ ] 实验员电脑（运行 study2_app 后端 + 前端）
- [ ] 实验员显示器（实验员控制面板，参与者不可见）
- [ ] 平板 / iPad（偏好表单 + 问卷，充满电）
- [ ] 机器人（TTS 音箱 + 机械臂，已完成调试）
- [ ] 录音设备（退出访谈录音，测试可用）
- [ ] 物理场景道具（三个 trial 所需的物品各 16 件 + 容器标签）
- [ ] 知情同意书（纸质，含录音授权条款）
- [ ] 参与者编号对照表（已打印，含 Latin Square 分配）

### 1.2 系统启动

```bash
# 1. 激活 conda 环境
conda activate behavior

# 2. 配置 LLM 后端
export LLM_BACKEND=openai
export LLM_MODEL=gpt-5-chat
export LLM_API_KEY=YOUR_KEY
export LLM_BASE_URL=https://your-endpoint/v1

# 3. 启动后端
cd AskThenRearrange/study2_app/backend
python main.py

# 4. 启动前端（另开终端）
cd AskThenRearrange/study2_app/frontend
npm run dev

# 5. 浏览器打开实验员控制面板
# http://localhost:5173/experimenter
```

### 1.3 系统自检（参与者到达前 10 分钟）

- [ ] TTS 测试：点击控制面板测试按钮，确认机器人正常发声，音量适中
- [ ] STT 测试：按住录音按钮说一句话，松开后确认转录文字正确显示
- [ ] 平板测试：打开偏好表单页面，确认拖放功能正常
- [ ] 确认当前参与者的 Latin Square 行号，核对 Trial 1/2/3 的策略和场景顺序
- [ ] 将练习场景道具摆放就绪

### 1.4 参与者接待与知情同意

1. 引导参与者就座，简要介绍实验环境
2. 请参与者阅读知情同意书并签字（含录音授权）
3. 将平板递给参与者，填写**人口统计学问卷 + ATI + NARS**（约 8 分钟）
4. 回收平板

### 1.5 口头介绍（逐字稿）

> "感谢参与本研究。今天你会体验一个家居整理机器人，它会通过提问来了解你的整理习惯。你只需要用正常说话的方式回答机器人的问题就可以了。
>
> 每轮实验开始时，你会看到一个场景里的物品和可以放置它们的容器。然后机器人会向你提问，你口头回答。机器人会根据这段对话，对场景里**所有物品**做出整理预测——包括它没有直接问过你的物品。当你觉得机器人已经足够了解你的整理方式了，直接告诉我'可以结束了'，我来操作结束这个对话。
>
> 对话结束后，你需要在平板上为每件物品选择你最希望它被放置的容器，再完成一个简短的问卷。最后我们会一起看看机器人的预测结果。整个过程大约需要 65 分钟。
>
> 在正式开始前，我们先用一个练习场景熟悉一下流程。有任何问题都可以随时问我。"

**注意**：全程不提及"策略"或"条件"，不告知参与者存在三种不同的提问方式。

### 1.6 练习 Trial

**场景**：练习场景（不纳入分析）｜**策略**：UPF（固定）

1. 在控制面板选择 **Practice Trial**，策略选 UPF，点击 **Start Scene**
2. 摆好练习场景道具，将平板递给参与者（显示物品网格）
3. 按照正式 trial 的完整流程走一遍 Phase 1–5（步骤见第二节）
4. 练习结束后询问参与者：**"整个流程清楚了吗？有什么问题吗？"**

**练习阶段常见问题回答**：

| 参与者问题 | 实验员回答 |
|---|---|
| "我应该怎么回答？" | "按照你平时的整理习惯回答就好，没有标准答案。" |
| "我能反悔或者纠正吗？" | "可以，在对话阶段直接告诉机器人你想修改就好。" |
| "需要回答多详细？" | "你觉得合适就好，可以简单也可以详细。" |

## 二、实验中

以下操作针对**每一个正式 trial**（共 3 个），按 Phase 顺序执行。

### Phase 1：场景展示（约 3 分钟）

**助手操作**：
1. 在控制面板选择当前 trial（Trial 1/2/3），确认策略和场景已按 Latin Square 填入
2. 点击 **Start Scene**

**主实验员操作**：
1. 将当前场景的物理道具摆放在桌面上
2. 将平板递给参与者（显示 16 件物品网格）
3. 对参与者说：
   > "请先看一下桌上的物品和标有名字的容器，同时参考平板上的清单。熟悉之后告诉我准备好了。"
4. 参与者说准备好后：收回平板，助手点击 **Advance to Dialogue**

---

### Phase 2：对话（时长不定，参与者自主控制）

> ⚠️ **本阶段参与者不看屏幕，只与机器人交互。**

**每轮（turn）操作循环**（助手执行）：

1. 点击 **Generate Question** → 机器人自动播放 TTS 问题
2. 等待机器人说完，参与者开始口头回答
3. **按住 STT 录音按钮**，录制参与者完整回答；听到回答结束后松开
4. 系统返回转录文字，显示在文本框中
5. 如有明显转录错误，快速手动修正（**不得改动语义，只修正识别错误**）
6. 点击 **Submit Answer**
7. 侧边栏自动更新当前 PSR@k（仅实验员可见，**不告知参与者**）
8. 重复以上步骤，直到参与者主动表示结束

**参与者说"可以结束了"（或类似表达）时**（助手执行）：
- 点击 **End Dialogue** → 系统触发 Final Placement Planner，自动记录 turn_count

**注意事项**：
- 不主动暗示参与者"还可以继续"或"可以结束了"——让参与者完全自主决定
- 若 TTS 播放失败：助手口头朗读屏幕上的问题原文，继续录音流程
- 若 STT 严重失误（3 处以上实质错误）：告知参与者"我帮你重新录一下可以吗？"，重新录音后替换

---

### Phase 3：偏好表单（约 5 分钟）

**助手操作**：点击 **Advance to Preference Form**

**主实验员操作**：
1. 将平板递给参与者（显示偏好表单）
2. 对参与者说：
   > "请在平板上为每一件物品选择你最希望它被放置的容器。不需要考虑机器人刚才的问题，按照你自己真实的整理习惯来填写。所有 16 件物品都选完之后才能提交。"
3. 参与者提交后收回平板

---

### Phase 4：问卷（约 5 分钟）

**助手操作**：点击 **Advance to Questionnaire**

**主实验员操作**：
1. 将平板递给参与者（依次显示 NASA-TLX → 感知理解问卷 → 交互主导感问卷）
2. 对参与者说：
   > "请完成这份关于刚才对话体验的问卷，根据你的真实感受作答就好。"
3. 参与者提交后收回平板

> ⚠️ **此时绝对不展示系统预测结果。** 问卷的有效性依赖于参与者对准确率"完全不知情"。

---

### Phase 5：系统预测展示（约 2 分钟）

**助手操作**：点击 **Show Prediction**（屏幕显示系统预测 vs 参与者偏好对比）

**主实验员操作**：
1. 将屏幕转向参与者
2. 对参与者说：
   > "这是机器人根据你刚才的回答，对每件物品的放置预测。绿色是和你填的一致的，红色是不一致的。"
3. 给参与者 1–2 分钟浏览；可以简单回应反应，**不展开讨论**（讨论留到退出访谈）

**Phase 5 结束后**：
- Trial 1 / Trial 2 结束：助手点击 **Next Trial**，进入 trial 间隔（见 2.6 节）
- Trial 3 结束：进入退出访谈（见第三节）

---

### 2.6 Trial 间隔（约 3 分钟）

**主实验员**：对参与者说 > "我来换一下场景，你休息一下，大约 3 分钟。"

**助手同时**：
- [ ] 撤去旧场景道具，摆出下一个场景道具
- [ ] 在控制面板配置下一 trial 的场景信息（策略 + 场景名）
- [ ] 确认策略与 Latin Square 一致

---

### 突发情况处理

| 情况 | 处理方式 |
|---|---|
| **机器人 TTS 失败** | 助手口头朗读屏幕上的问题原文，继续录音流程 |
| **STT 严重失误（3+ 处实质错误）** | 告知参与者重新录音，替换转录文字 |
| **参与者长时间沉默（>20 秒）** | 主实验员轻声提示："可以用你平时会怎么做来回答，不用太正式。" |
| **参与者想修改之前的回答** | 在当前 turn 文本框末尾追加补充说明后提交，例如："（补充：之前说的书放书架，我更希望放桌面。）" |
| **后端崩溃 / 系统报错** | 记录当前 turn_index，重启后端，从 session 恢复页面继续 |
| **参与者问"机器人表现好不好"** | "我们在问卷之后会给你看结果，先把问卷填完。"（Phase 5 前不透露） |
| **参与者中途退出** | 立即停止录音，导出当前数据，标注为 incomplete，不纳入分析 |

## 三、实验后

### 3.1 策略偏好排序

**助手操作**：点击 **Advance to Final Ranking**，将平板递给参与者

**主实验员**：
> "回顾一下你刚才体验的三种对话方式，请按照你最喜欢的顺序排一下。没有对错之分，完全按主观感受来。"

- 参与者完成排序后，点击 Optional Comment 让参与者填写补充意见（可选）
- 助手点击 **End Session**，系统保存数据

### 3.2 退出访谈（约 12 分钟）

**开启录音设备**（再次口头确认参与者同意录音）。主实验员主导，助手做笔记。

**A. 整体感受**
- "三种对话方式整体感觉怎么样？有没有让你印象深刻的地方？"

**B. 比较感知**
- "三种方式中，哪种感觉最自然？为什么？"
- "哪种感觉最费力，或者最难回答？"
- "哪种方式让你觉得机器人最'理解'你的整理习惯？"

**C. 问题感知**
- "有没有某些问题让你觉得奇怪、多余或者难以回答？"
- "有没有某些问题让你觉得特别有帮助？"

**D. 预测结果**
- "看到机器人的预测，有没有让你意外的地方？"
- "你觉得机器人真的理解了你的整理习惯吗？"

**E. 真实场景**
- "如果家里真的有这样一个机器人，你希望它用哪种方式向你提问？"
- "你觉得提多少个问题比较合适？"

**结束语**：
> "非常感谢你今天参与！你的反馈对我们的研究很有帮助。如有任何问题，可以通过邮件联系我们。"

### 3.3 数据导出

1. 在实验员控制面板点击 **Export Session Log**
2. 下载 JSONL 文件，命名格式：`P{编号}_{YYYYMMDD}.jsonl`（例：`P07_20260420.jsonl`）
3. 将文件复制到 `AskThenRearrange/logs/` 目录
4. 验证文件完整性：

```python
import json
with open("logs/P07_20260420.jsonl") as f:
    events = [json.loads(line) for line in f]
print(len(events), "events recorded")
```

5. 保存退出访谈录音文件，命名格式：`P{编号}_{YYYYMMDD}_interview.m4a`
6. 将 JSONL 和录音文件**当日备份至加密存储**
7. 点击控制面板 **Reset Session**，清空系统状态，准备下一次 session

## 伦理与盲法要求

1. **盲法**：Phase 4 问卷之前，绝对不透露任何关于预测准确率的信息
2. **不干预对话**：Phase 2 期间，实验员不对参与者的回答作任何评价或引导
3. **策略保密**：全程不告知参与者三种条件的名称或技术差异
4. **录音授权**：退出访谈开始前再次口头确认参与者同意录音
5. **随时可退出**：如参与者表示不想继续，立即停止，提供全额补偿
